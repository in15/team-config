/team-config

Assemble your
agent squad.

One command to configure an entire agent pipeline with role-based personas, sequential handoffs, and human checkpoints.

Drop one file into any repo. Run /team-config. Build.

THE PROBLEM

Agents are powerful.
But unstructured agents are chaos.

🚨 No clear handoff between "spec" and "code" and "review"
🔄 Agents redo each other's work or skip critical steps
💭 No record of why decisions were made
🔧 Setting up teams manually every time is tedious

THE SOLUTION

A structured pipeline
that runs itself.

1. Each agent has a specific role and a distinct persona
2. Stages run sequentially — each one feeds the next
3. You choose where to insert human checkpoints
4. A Scribe silently documents every decision and tradeoff
5. Save config, reload it next time with /team-config --load

THE SQUAD

Eight roles. Eight personas.

🏗
The Architect
Turns ideas into specs no one can misread
"You measure twice because cutting twice is expensive."
🔍
The Skeptic
Pokes holes in blueprints before code is written
"I'd rather find the holes now than in production at 2 AM."
⚒
The Test Smith
Writes tests first. That's the religion.
"A failing test is a promise."
🧪
The Test Critic
Makes sure the tests actually prove something
"The worst test is one that passes when the code is broken."
🔨
The Builder
Minimum code, maximum correctness
"Over-engineering is a bug."
🛡
The Gatekeeper
Nothing ships until this passes review
"'It works on my machine' means nothing."
⚖
The Judge
Final verdict: did we build what we promised?
"I'm not here to be impressed. I'm here to verify."
📝
The Scribe
Watches everything. Writes down what matters.
"Every word earns its place or gets cut."

REVIEW LOOPS + FRESH EYES

They iterate until it's right.
Then fresh eyes verify.

Phase A — In-Session Iteration

Architect writes spec
  ↓
Skeptic reviews → SENT BACK
  ↓
Architect fixes → re-submits
  ↓
Skeptic reviews → APPROVED

Same agent throughout. Full context. No limit on rounds.

Phase B — Fresh Eyes

New Skeptic spawns (zero context)
  ↓
Reads final spec cold
  ↓
"CONFIRMED. Fresh eyes found nothing new."
  ↓
Pipeline advances →

No anchoring. No baggage. Just: does this hold up?

● Architect ↔ Skeptic
● Test Smith ↔ Test Critic
● Builder ↔ Gatekeeper

THE SCRIBE

Every decision,
documented.

No fluff. No filler. No "in conclusion."

"Every word earns its place or gets cut."

Session Notes — Auth Refactor

Compiled by The Scribe

The Blueprint

Decision: JWT with refresh tokens | Why: Stateless scales better than sessions

Tradeoff: No instant revocation | OK because: 15-min expiry is acceptable

The Skeptic's Take

Flagged: No rate limiting on refresh | Resolution: Added to spec

The Build

Approach: Middleware + decorator pattern

Not this: Custom auth backend | Because: framework middleware is sufficient
